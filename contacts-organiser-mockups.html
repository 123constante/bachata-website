Event page · Contacts tile + Organiser routing

Today the bento tile ContactsBlock shows external links only (Website / Instagram / Facebook) and a separate EventOrganiserSection renders the host as its own tile. The brief: pull the organiser into the same tile so the user sees the host first and is routed to the on-platform profile, not bounced out to Instagram. Tokens below mirror --bento-* from src/index.css: forest-green surface, tarnished brass accents, aged-cream body. Sample organiser used throughout: Bos Dance.

AFeatured profile header

Organiser sits at the top of the Contacts tile in a slightly larger, brass-tinted row. Distinct from the external-link rows below it. Reads as the primary call-to-action.

Pros / cons

  • Highest signal: organiser is unmistakably the primary contact, not equal to IG.
  • Visual cue (brass tint) maps to Bento's accent token — feels native, not bolted on.
  • Good for the platform's stated routing goal (keep users on-site via organiser profile).
  • Tile gets ~32 px taller. On dense layouts this nudges the bento grid below.
  • Visual weight risks pulling the eye away from the ticket CTA above.
