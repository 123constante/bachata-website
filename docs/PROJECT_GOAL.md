# Project Goal — Public Website

## Purpose of this website

The purpose of this website is to make it easy for dancers to discover events anywhere in the world.

Users should be able to quickly understand:

what is happening  
where it is happening  
when it is happening  
who is involved  

with minimal effort.

The website exists to make event discovery fast and clear on mobile devices.

---

## Core principle

The website is for discovery, not editing.

Users should be able to:

browse events  
search events  
filter events  
open event details  
see session information  
see teachers, DJs, performers, and guests  

without friction.

---

## Mobile-first requirement

Most users will access the website from mobile devices.

Layouts must:

minimise scrolling  
keep important information visible immediately  
avoid large empty spacing  
keep session information compact  
display people clearly and consistently  
prioritise readability over decoration  

Desktop layout adapts from mobile layout, not the reverse.

---

## Event information clarity rule

Events must be understandable at a glance.

Users should immediately see:

event name  
date  
location  
session schedule  
who is teaching  
who is DJing  
who is performing  

Session-level people linking should always be reflected clearly on the event page.

---

## Terminology rule

Public users should see simple language.

Sessions should normally appear as:

Classes  
Parties  

More detailed session types (bootcamp, masterclass, training, etc.) may appear inside detailed views but should not make the interface harder to understand.

Clarity is more important than technical accuracy.

---

## Platform objective

The long-term objective of this platform is to list every dance event in the world.

The public website must support:

fast browsing  
clear filtering  
mobile usability  
city-level exploration  
festival schedule visibility  
session-level teacher and DJ visibility  

All future website decisions should prioritise:

clarity  
speed  
mobile usability  
global scalability  

Features that slow down event discovery should be avoided.
